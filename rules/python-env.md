从现在开始，你在处理本项目中的 Python 相关任务时，必须严格遵守以下环境规则。除非我明确批准，否则不得偏离。

==============================
一、总体原则
==============================

1. 你必须优先保证 Python 环境稳定、可复现、可追踪。
2. 不要为了临时运行代码而随意切换 Python 解释器、安装依赖或创建新环境。
3. 所有 Python 操作都必须优先基于当前项目已有环境。
4. 未经我明确确认，不要修改系统级 Python、全局 pip、全局 site-packages。
5. 未经我明确确认，不要使用 sudo 安装 Python 包。

==============================
二、解释器与环境选择规则
==============================

1. 在运行任何 Python 命令之前，必须先检查并报告当前使用的 Python 环境。
2. 你必须先输出以下信息，再继续执行：
   - python 可执行文件路径
   - python 版本
   - pip 可执行文件路径
   - 当前虚拟环境名称（如果有）
   - 项目中是否存在 .venv、venv、conda 环境、poetry、uv、requirements.txt、pyproject.toml

3. 优先级如下：
   - 项目明确指定的虚拟环境
   - 项目根目录下的 .venv
   - 项目已有 venv
   - 项目明确指定的 conda 环境
   - 若都不存在，先询问我，不要自行创建新环境

4. 不要擅自切换到系统 Python。
5. 不要因为命令失败就自动换另一个 Python 解释器重试。
6. 不要混用 python、python3、pip、pip3，必须明确对应同一个解释器。

==============================
三、依赖安装规则
==============================

1. 未经我确认，不要执行任何安装命令，包括但不限于：
   - pip install
   - pip3 install
   - python -m pip install
   - uv pip install
   - poetry add
   - conda install

2. 如果确实需要安装依赖，你必须先输出：
   - 需要安装的包名
   - 安装原因
   - 安装到哪个环境
   - 是否会修改 requirements.txt / pyproject.toml / lock 文件
   - 可能的兼容性风险

3. 未经我确认，不要安装：
   - 最新版不锁版本的包
   - 全局包
   - 与现有依赖冲突风险高的包

4. 优先使用项目现有依赖管理方式：
   - requirements.txt
   - pyproject.toml
   - poetry.lock
   - uv.lock
   - conda env 文件

5. 不要为了跑通一个脚本而临时安装未记录依赖却不更新依赖文件。

==============================
四、运行 Python 脚本规则
==============================

1. 运行任何 Python 脚本前，必须先说明：
   - 使用哪个解释器
   - 使用哪个环境
   - 运行的命令
   - 脚本入口文件
   - 预期作用
   - 是否会写文件、改数据、联网、下载模型、占用 GPU/大量 CPU

2. 对于有副作用的脚本，必须先提醒我确认。
   有副作用包括但不限于：
   - 写文件
   - 改配置
   - 访问数据库
   - 调用外部 API
   - 下载数据/模型
   - 删除缓存
   - 训练任务
   - 长时间运行任务

3. 默认优先执行：
   - 只读检查
   - 小范围测试
   - dry-run
   - 单元测试
   而不是直接跑完整任务。

4. 如果只是做语法或导入检查，优先选择最小化验证方式，不要直接启动整个系统。

==============================
五、环境创建与修改规则
==============================

1. 未经我确认，不要创建新的 Python 虚拟环境。
2. 未经我确认，不要删除、重建、覆盖现有 .venv 或 conda 环境。
3. 未经我确认，不要升级 pip、setuptools、wheel。
4. 未经我确认，不要修改：
   - requirements.txt
   - pyproject.toml
   - poetry.lock
   - uv.lock
   - environment.yml
   - .python-version

5. 如果你判断当前环境损坏或不一致，必须先报告问题，再给出修复建议，不要直接重建。

==============================
六、版本与兼容性规则
==============================

1. 你必须优先遵循项目已有 Python 版本约束。
2. 如果项目中存在以下任一文件，必须优先读取并遵守：
   - .python-version
   - pyproject.toml
   - requirements.txt
   - environment.yml
   - README 中的环境说明

3. 如果发现当前解释器版本与项目要求不一致，必须先报告，不要硬跑。
4. 不要默认升级到最新 Python 版本。
5. 对于 Apple Silicon、Linux、macOS 等平台兼容性问题，必须明确指出风险。

==============================
八、禁止事项
==============================

未经我明确允许，你禁止：

1. 使用系统全局 pip 安装包
2. 使用 sudo pip install
3. 自动创建新虚拟环境
4. 自动切换 Python 版本
5. 自动修改依赖文件
6. 自动下载大型模型或数据集
7. 自动启动长时间训练任务
8. 自动运行会写文件或改数据的脚本
9. 混用多个 Python 环境而不说明
10. 命令失败后擅自换环境重试

==============================
九、推荐行为
==============================

你应该优先：
1. 先识别项目现有环境
2. 先报告环境状态
3. 先做最小化验证
4. 先征求我确认，再执行有副作用操作
5. 让所有 Python 操作可复现、可记录、可回滚

从现在开始，所有 Python 相关操作都必须严格遵守以上规则。
